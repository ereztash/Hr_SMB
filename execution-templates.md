## Input:
- Current perception among peers
- Natural strengths

## Process:

### Step 1: Mega-Trait Discovery

**User Task**: Complete this sentence 5 ways:
1. "People often say I'm unusually good at: ___"
2. "The one thing clients thank me for most is: ___"
3. "In my industry, I'm known for: ___"
4. "My closest friend would say my superpower is: ___"
5. "If I had to pick ONE thing that differentiates me, it's: ___"

**Agent Analysis**:
- Look for commonality across answers
- If scattered = no clear mega-trait
- If consistent = found it

### Step 2: Halo Amplification Strategy

**Once Mega-Trait Identified**, create strategy to amplify it:

| Channel | How to Amplify Mega-Trait |
|---------|---------------------------|
| **LinkedIn** | Lead with it in: headline, first sentence of profile, first post topic |
| **Speaking** | Open with it: "I'm known for [trait], here's why it matters..." |
| **Content** | 70% of content should directly showcase this trait |
| **First Meeting** | Mention it in first 5 minutes naturally |
| **Portfolio** | Show 3-5 best examples of this trait in action |
| **Testimonials** | Ask clients to mention this specific trait |

### Step 3: Horn Effect Audit (If Exists)

**User Task**: 
"Is there ONE negative perception that overrides the halo?"

Examples:
- Negative: "Too young" / "Too old" / "Job-hopped" / "Failed visibly"

If yes:
- Priority #1: Fix the horn BEFORE amplifying halo
- Strategy: Address it proactively in first meeting/profile
- Timeline: Can take 6-12 months to recover from horn effect

# TEMPLATE 4: LOSS AVERSION REFRAMING

## Purpose:
×¢×–×¨ ×œ××§×¦×•×¢ ×œ×”×ª×’×‘×¨ ×¢×œ status quo bias ×•loss aversion ×œreposition ××©××¢×•×ª×™.

## Input:
- Current fears about repositioning (from Diagnostic)
- Actual vs. anticipated losses
- Time horizon

## Process:

### Step 1: Fear Inventory

**User Task**: "List 3-5 fears about repositioning:"
1. [Fear 1]: "I'll lose my credibility in my current niche"
2. [Fear 2]: "My network won't understand the shift"
3. [Fear 3]: "It will take months with no income"
4. [Fear 4]: "I'm not sure if the new positioning is even real"
5. [Fear 5]: [Your fear]

### Step 2: Reality Testing Each Fear

**For Each Fear**, Agent runs:

**Question Pattern**:
1. "What's the WORST case scenario for this fear?"
   - Record actual outcome, timeline, impact

2. "How likely is this worst case?" (probability %)
   - Scoring: <10% = unlikely; 50%+ = real risk

3. "If it happened, what could you do?"
   - Identify recovery options (pivot back? adjust? continue?)

4. "How many people have repositioned successfully despite this risk?"
   - Identify counterexamples

**Example - Fear: "I'll lose my credibility"**
```
Worst case: Current network sees me as confused/fickle
Probability: 30%
Recovery: Can explain evolution, not pivot; highlight niche deepening
Counter-examples: 10+ examples of successful niche narrowing (actually improved perception)
```

### Step 3: Reframing the Repositioning

**Offer 3 Framing Options**:

**Framing A: "Evolution"**
- "I'm not pivoting away from expertise, I'm going deeper into the core of what I do"
- Loss language: Minimal
- Best for: When repositioning is niche-deepening

**Framing B: "Reclamation"**
- "I've always been this way, I'm finally claiming it"
- Loss language: Minimal
- Best for: When you're returning to roots

**Framing C: "Experiment"**
- "I'm testing positioning for 90 days; we'll see if it resonates"
- Loss language: Temporary, reversible
- Best for: When significant change, want low commitment

**User Choice**: Pick framing that feels most authentic

### Step 4: Micro-Commitment Strategy

**Small Steps Over Large Leap**:

Instead of: Announce new positioning to everyone at once
Do: Phase it in via:

- **Week 1**: Update LinkedIn headline only
- **Week 2**: Test new positioning with 3 warm contacts
- **Week 3**: Write 1 article in new framing
- **Week 4**: Use new positioning in 5 new conversations
- **Week 5**: Update full LinkedIn profile
- **Month 2+**: Build social proof in new positioning

This reduces loss aversion by:
- Testing before full commitment
- Getting feedback early
- Allowing reversal if needed
- Building evidence as you go

```
- Time to build: 5-10 hours

- Reusability: Medium

```

- Credibility: Medium-low
- Reusability: Medium

- **LinkedIn**: Post it as article (get views, engagement)
- **Website**: Feature prominently
- **Outreach**: Link it in cold emails to prospects
- **Social Proof**: Use in pitches to new clients

### Step 4: Proof Flywheel

After first proof:
- **Week 2**: "If I helped Client A with X, I can likely help others with X. Interested in exploring?"
- **Week 3**: Land Client B (has strong proof that it works)
- **Week 4**: Build proof for Client B (now you have 2)
- **Week 5+**: Each new client = 1 new proof point; flywheel accelerates

- You host office hours, AMA, events
- You curate content
- You create structure/value for the community

**Simple Actions to Shift**:
1. Start: "I'm noticing [problem] in our community. Want to solve it together?"
2. Propose: "I'd like to run a weekly [discussion/office hours/workshop]"
